Fetchr.js Documentation

Fetchr is an NPM package for the simple creation of global requests. You enter all requests in a YAML file and can access them in the entire project. Fast, simple and without complications. If there are any bugs please report them in our GitHub Repository

Get Started

Prerequisites

Before you begin, you will need the following:

  • Node.js LTS: Install the latest stable LTS version of Node.js. You can find information on the Node.js website: https://nodejs.org/en/download.

1. Create a project

Create a new directory for your project and navigate to that directory. Initialize a new npm project with the following command:

npm init

This will create a file called package.json which contains information about your project.

2. Install dependencies

Install the fetchr.js library with the following command:

npm install fetchr.js

3. Create a configuration file

Create a file called fetchr.yml in the root directory of your project. This file will contain the configuration for fetchr.js.

  • You can freely decide the name and location of the file. Example: config/routes.yml
Example:
config:
  baseURL: "https://api.example.com"
  debug: false
  errors: false
  defaultHeaders: {
	  "Content-Type": "application/json",
  }

routes:
  test:
	params: ["token"]
	endpoint: "/example"
	method: "GET"
	headers: {
		"Authorization": "Bearer {token}"
	}

4. Next steps

Once you have completed the basic steps, you can start using fetchr.js.

Configuration

General Configuration

These are general settings that are set under the "config" item

Key Type Description Default Required
baseURL string The main URL where the resources can be found null Yes
debug boolean If true then all requests are printed to the console false No
errors boolean If true then errors are printed detailed to the console false No
defaultHeaders json These headers are attached to each request {} No

Routes Configuration

Routes configurations are used for requests

Key Type Description Default Required
url string The endpoint at which the resource is located null Yes
body json Any body that you want to add to your request null No
method string The request method, e.g., GET, POST GET No
cache string The cache mode you want to use for the request. default No
headers json Any headers you want to add to your request {} No
mode string The mode you want to use for the request, e.g., cors, no-cors, same-origin, or navigate cors No
redirect string The redirect mode to use: follow, error, or manual follow No
